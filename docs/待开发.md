    - 现象：apinova.cc 中转 GPT-5.5 时，第 2 轮请求返回 400

      没透传 `delta.tool_calls[i].function.name`，导致 acecode 收到
      tool_calls 但 function_name 为空（日志：`WRN Unknown tool: `），
      acecode 把空 name 的 tool_call 写回 messages_，下一轮回传被 400
    - 复现日志：N:\Users\shao\dbcoding4\acecode.log
    - 兜底方案：openai_provider.cpp 的 SSE flush 处（约 270-284、382-397
      两处）发现 `tc.name.empty()` 时丢弃该 tool_call 并 LOG_ERROR；
      若一轮内全部被丢弃，按 text-only 结束 loop，避免毒数据进历史
